### 1.2 技术栈选型

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| **前端** | React 18 + TypeScript 5 | 主流框架，类型安全 |
| **状态管理** | Zustand + React Query | 轻量级状态管理，服务端状态缓存 |
| **UI组件** | Ant Design 5 | 企业级组件库 |
| **图表** | ECharts 5 | 数据可视化 |
| **后端** | NestJS 10 + TypeScript | 企业级Node.js框架 |
| **数据库** | PostgreSQL 16 | 关系型数据库，支持JSON |
| **缓存** | Redis 7 | 高性能缓存 |
| **消息队列** | RabbitMQ | 异步消息处理 |
| **文件存储** | MinIO | 对象存储 |
| **网关** | Nginx | 反向代理、负载均衡 |
| **容器** | Docker + Docker Compose | 容器化部署 |

## 6. 安全设计

### 6.1 认证授权流程

```
┌─────────┐                              ┌─────────────┐
│  客户端  │ ──(1) 用户名+密码──────────▶ │  认证服务   │
│         │                              │             │
│         │ ◀─(2) AccessToken+Refresh─── │             │
│         │                              └─────────────┘
│         │
│         │ ──(3) AccessToken(API请求)──▶ ┌─────────────┐
│         │                              │  API服务    │
│         │                              │  - 验证Token │
│         │ ◀─(4) 受保护资源───────────── │  - 权限检查  │
│         │                              └─────────────┘
│         │
│         │ ──(5) RefreshToken──────────▶ ┌─────────────┐
│         │                              │  刷新Token   │
│         │ ◀─(6) 新AccessToken────────── │             │
└─────────┘                              └─────────────┘
```

### 6.2 安全措施

| 安全层 | 措施 |
|--------|------|
| 传输安全 | HTTPS/TLS 1.3，证书固定 |
| 接口安全 | JWT Token，过期时间15分钟，Refresh Token 7天 |
| 权限控制 | RBAC模型，菜单级+按钮级+数据级权限 |
| 输入验证 | 参数校验、SQL注入防护、XSS防护 |
| 敏感数据 | 密码BCrypt加密，关键数据AES加密存储 |
| 操作审计 | 全量操作日志，敏感操作二次确认 |
| 访问控制 | IP白名单、登录失败锁定、异地登录提醒 |

## 8. 扩展性设计

### 8.1 水平扩展

- **无状态服务**：API服务无状态设计，支持水平扩展
- **负载均衡**：Nginx轮询 + 健康检查
- **会话共享**：JWT Token替代Session，Redis存储临时状态

### 8.2 数据库扩展

- **读写分离**：主库写，从库读
- **分库分表**：按业务模块分库，按时间分表
- **归档策略**：历史数据定期归档

### 8.3 缓存策略

| 数据类型 | 缓存策略 | 过期时间 |
|----------|----------|----------|
| 实时行情 | Redis Pub/Sub | 实时 |
| 用户会话 | Redis | 15分钟 |
| 权限数据 | Redis + 本地缓存 | 1小时 |
| 报表数据 | Redis | 1小时 |
| 基础数据 | Redis | 24小时 |
