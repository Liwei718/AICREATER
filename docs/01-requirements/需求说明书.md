## 1. 项目概述

### 1.1 项目背景
本项目为一家**有色金属（铜）生产企业**定制开发套期保值管理系统。作为铜生产企业，公司面临着铜价波动带来的经营风险，需要通过期货市场进行卖出套期保值，锁定产品销售价格，稳定经营利润。

### 1.2 项目目标
构建一套覆盖套保业务全链路的管理系统，实现：
- 头寸全生命周期管理
- 套保方案制定与审批
- 交易执行与跟踪
- 实时风险监控预警
- 套保效果量化分析
- 财务报表自动生成
- 合规性管控

### 1.3 业务范围
| 项目 | 说明 |
|------|------|
| 套保品种 | 铜期货（CU）、铜期权 |
| 套保方向 | 卖出套期保值为主 |
| 交易市场 | 上海期货交易所（SHFE） |
| 涉及币种 | 人民币（CNY） |

## 3. 功能需求

### 3.1 头寸管理模块

#### 3.1.1 现货头寸管理
**功能描述**: 管理企业的现货库存、销售订单、采购订单等现货敞口

**核心功能**:
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 库存录入 | 录入铜库存（原料、在制品、产成品），含数量、成本、存放地点 | P0 |
| 销售订单 | 录入销售合同/订单，含数量、交货期、定价方式（点价/均价/固定价） | P0 |
| 采购订单 | 录入原料采购订单，含数量、交货期、定价方式 | P0 |
| 现货敞口计算 | 自动计算净现货敞口 = 库存 + 已采购未到货 - 已销售未交货 | P0 |
| 点价管理 | 记录点价指令、点价价格、点价日期，关联销售合同 | P1 |
| 库存预警 | 库存低于/高于设定阈值时预警 | P1 |

**数据字段**:
- 库存：品种、数量（吨）、单位成本、存放仓库、入库日期、品质等级
- 销售订单：合同号、客户、数量、单价类型、基准价、升贴水、交货期、点价状态
- 采购订单：合同号、供应商、数量、单价类型、交货期

#### 3.1.2 期货头寸管理
**功能描述**: 管理期货合约持仓、开仓平仓记录

**核心功能**:
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 持仓录入 | 手工录入期货持仓或从成交记录自动生成 | P0 |
| 持仓查询 | 按合约、方向、开仓日期等多维度查询持仓 | P0 |
| 持仓调整 | 支持持仓合并、拆分、移仓换月 | P1 |
| 交割管理 | 期货交割意向登记、交割配对跟踪 | P1 |
| 持仓到期预警 | 合约到期前N日预警提醒 | P1 |

**数据字段**:
- 持仓：合约代码、买卖方向、持仓量、开仓均价、开仓日期、保证金占用、盈亏

### 3.2 套保方案模块

#### 3.2.1 套保方案制定
**功能描述**: 制定套期保值方案，确定套保比例、合约选择、执行策略

**核心功能**:
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 敞口识别 | 系统自动识别待保值现货敞口 | P0 |
| 方案创建 | 选择保值标的、保值比例（%）、目标合约、计划数量 | P0 |
| 策略配置 | 设置建仓方式（一次性/分批）、价格区间、止盈止损 | P1 |
| 基差分析 | 显示历史基差走势，辅助决策 | P1 |
| 方案模拟 | 模拟不同价格情景下的套保效果 | P2 |

**审批流程**:
```
交易员创建 → 部门经理初审 → 风控部审核 → 公司领导审批 → 生效执行
```

#### 3.2.2 方案执行跟踪
**功能描述**: 跟踪套保方案的执行进度

**核心功能**:
- 方案执行进度（已建仓数量/计划数量）
- 方案与实际持仓对比
- 方案调整/终止申请
- 方案执行报告

### 3.3 交易执行模块

#### 3.3.1 交易指令管理
**功能描述**: 期货交易指令的创建、审批、执行

**核心功能**:
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 指令创建 | 创建开平仓指令，关联套保方案 | P0 |
| 指令审批 | 按额度设置审批流程 | P0 |
| 指令分配 | 将指令分配给具体交易员执行 | P0 |
| 指令状态跟踪 | 待审批/已批准/已执行/已撤销 | P0 |
| 指令风控检查 | 超额度、超权限自动拦截 | P0 |

**指令类型**:
- 开仓指令：买入/卖出开仓
- 平仓指令：买入/卖出平仓
- 移仓指令：平仓近月+开仓远月

#### 3.3.2 成交记录管理

**核心功能**:

### 3.4 风险监控模块

#### 3.4.1 实时风险监控
**功能描述**: 实时监控套保业务各项风险指标

**核心功能**:
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 风险度计算 | 风险度 = 保证金占用 / 权益 × 100% | P0 |
| 保证金监控 | 实时显示各账户保证金占用、可用资金 | P0 |
| 涨跌停监控 | 持仓合约涨跌停板风险预警 | P0 |
| 交割月监控 | 持仓进入交割月前预警 | P0 |
| 集中度监控 | 单一合约/品种持仓集中度 | P1 |
| VaR计算 | 在险价值计算 | P2 |

**预警规则**:
| 指标 | 提醒阈值 | 预警阈值 | 告警阈值 |
|------|----------|----------|----------|
| 风险度 | 60% | 80% | 90% |
| 可用资金 | 100万 | 50万 | 20万 |
| 单合约亏损 | - | -5万 | -10万 |

#### 3.4.2 套保合规监控
**功能描述**: 监控套期保值业务的合规性

**核心功能**:
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 套保额度监控 | 已用额度/获批额度实时监控 | P0 |
| 建仓比例监控 | 期货/现货比例是否超标 | P0 |
| 方向一致性检查 | 检查期货方向与现货敞口方向是否匹配 | P0 |
| 投机持仓检查 | 识别超出套保范围的投机持仓 | P0 |
| 报告提醒 | 交易所套保报告提交提醒 | P1 |

### 3.5 盈亏分析模块

#### 3.5.1 期现结合盈亏分析
**功能描述**: 综合分析期货和现货的合并盈亏

**核心功能**:
| 功能点 | 描述 | 优先级 |
|--------|------|--------|
| 期货盈亏 | 按日、周、月统计期货盈亏 | P0 |
| 现货盈亏 | 统计已实现/未实现现货盈亏 | P0 |
| 套保效果 | 计算套保比率、套保有效性 | P0 |
| 基差盈亏 | 基差变动对套保效果的影响分析 | P1 |
| 盈亏归因 | 盈亏来源分解（价格、基差、数量等） | P1 |

**关键指标**:
- **套保比率** = 期货盈亏 / 现货盈亏 × 100%
- **套保有效性** = (现货盈亏 + 期货盈亏) / |现货盈亏| × 100%
- **基差盈亏** = (建仓基差 - 当前基差) × 数量

#### 3.5.2 多维度分析报表
**功能描述**: 提供灵活的盈亏分析报表

**分析维度**:
- 按时间：日报、周报、月报、季报、年报
- 按品种：铜期货、铜期权细分
- 按方案：分方案统计盈亏
- 按部门/交易员：业绩统计
- 按合约：分合约盈亏统计

### 6.1 术语表
| 术语 | 解释 |
|------|------|
| 套期保值 | 在期货市场上建立与现货市场相反的头寸，以对冲价格风险 |
| 基差 | 现货价格 - 期货价格 |
| 卖出套保 | 持有现货或预期销售，在期货市场卖出期货合约 |
| 点价 | 在基准价格基础上加上升贴水确定最终交易价格 |
| CTP | 综合交易平台，期货行业主流交易系统 |
| VaR | 在险价值，衡量潜在最大损失的风险指标 |
